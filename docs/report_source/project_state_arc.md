## 1. Summary

The teammate's variant adapts the original Hu et al. **ADAS** (Automated Design of Agentic Systems) framework to the **ARC-AGI** benchmark under a strict single-turn constraint. It is an **External, Self-identical** meta-agent system: a meta-agent re-designs candidate agents across iterations, then each candidate is evaluated on ARC tasks in a frozen downstream loop. The variant departs from the original ADAS in three significant ways (single-turn interception, dynamic mini-batch training, and a corrected train/eval split) and produces a set of ablation results that inform the broader meta-agent design discussion.

## 3. Three Key Contributions

### 3.1 Strict Single-Turn Interception

Original ADAS permits candidate agents to run multi-turn tool loops. The teammate's variant enforces a **single inference call per ARC test case**, implemented at the **hardware/runtime level** (not via a soft prompt constraint) using a `thread_local.call_count` counter. After the first LLM call the second call raises, guaranteeing that any multi-turn pattern generated by the meta-agent collapses to its first response.

Observed effects:

- **Latency per task reduced ~74%** relative to unbounded multi-turn.
- Eliminates pathological "debug-loop" failure modes (e.g. the Coder paradigm's `transform(grid)` infinite-rewrite attempts observed at 29/100 in one run).
- Makes the evaluation axis (quality of a single-shot plan) directly comparable across candidate agents.

### 3.2 Dynamic Mini-Batch Sampling

Original ADAS scores each candidate on the full training set every iteration, which (a) is expensive and (b) encourages meta-overfitting to the fixed training set. The teammate samples **30 of the 100 training cases per iteration**, resampled each iteration.

Observed effects:

- Meaningful reduction in meta-overfitting (candidates generalise better to the held-out evaluation set).
- Multiplicative interaction with warm-starting from prior iterations: warm-start alone or mini-batch alone under-performs both combined.

### 3.3 Corrected Train / Eval Split

The original ADAS protocol has ambiguity between tuning the meta-agent on a "training" set and scoring the final candidate. The teammate's variant enforces a clean split: the 100-case **Evaluation Set is held out completely** until the chosen candidate is scored, producing a true zero-shot score rather than a train-adjacent one.

## 5. Negative Results and Design Discoveries

### 5.1 Coder paradigm collapses under single-turn

The Coder paradigm (agent emits a Python `def transform(grid)` that is executed against the test grid) **requires multi-turn debug loops** to be viable, because the first code attempt almost always has a bug. Under strict single-turn interception the debug loop is removed and the paradigm collapses to near-random performance. This is a direct consequence of the single-turn constraint rather than a failure of the paradigm per se, and it motivates the No-Coder (direct grid output) design as the appropriate pairing for single-turn evaluation.

### 5.2 DeepSeek-7B Semantic Deadlock (0/100)

DeepSeek-7B scored **0/100** across multiple configurations. Inspection of the outputs shows a systematic **"Semantic Deadlock"**: the model repeatedly produces plausible-looking reasoning about the task, but the final grid it emits is either malformed or unchanged from the input. This does not appear to be a prompt issue — the same prompts yield non-zero scores on Qwen — and is consistent with the model lacking the spatial-reasoning capacity ARC-AGI requires, not with any meta-agent defect. Reporting this negative result is useful context for the lit review's discussion of model-capability floors below which meta-agent optimisation cannot compensate.

### 5.3 Spontaneous complexity escalation

Across iterations, candidate agents produced by the meta-agent tend to **spontaneously grow in prompt length and branching complexity** even when simpler candidates scored similarly. The ablation suggests that this escalation is correlated with marginal training-set improvements but does **not** transfer to the held-out evaluation set — a form of meta-overfitting that the dynamic mini-batch mitigates but does not fully eliminate.

### 5.4 Multiplicative interaction: warm-start × mini-batch

Neither dynamic mini-batch nor warm-starting alone matches the combined configuration. This is consistent with the interpretation that (a) mini-batch provides the variance needed to escape the training-set optimum, while (b) warm-start supplies the prior structure that the next iteration refines. Either component alone leaves a gap; together they produce the best observed eval score.

## 6. Three Critiques of Original ADAS (as raised by the teammate)

1. **Unbounded multi-turn conflates agent design quality with debug-loop quality.** The original ADAS scores a candidate on how well its multi-turn behaviour eventually converges, not on the quality of its single-shot plan. This confounds the meta-optimisation signal.
2. **Full-set training encourages meta-overfitting.** Scoring every candidate on the full training set biases the meta-agent toward training-set idiosyncrasies rather than transferable agent designs.
3. **Train/eval ambiguity inflates reported scores.** Without a strict held-out evaluation set, ADAS numbers risk being train-adjacent rather than zero-shot, exaggerating the framework's apparent generalisation.

Each critique is matched one-to-one to one of the three contributions in §3.
